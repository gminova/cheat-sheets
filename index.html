    CSS

    RULE PRIORITY (weak to strong) COMMENT
    * The universal selector and inherited values have a specificity of 0 (e.g. *, body and similar) have a zero specificity.
    type/attribute selector {property:value;} e.g. h1, p, etc.
    From external CSS file: h1 {color: red;}

    In HTML file: h1 {color: yellow;}     		Contextual selectors are more specific than a single element selector.
    .class {property:value;} Often used for styling.
    #id {property:value;} Often used for JS functions.
    type selector(e.g. h1) {property:value !important;} Adding !important is useful to prevent libraries, like Bootstrap overriding important rules.
    .class {property:value !important;} .class is stronger than type selector if both are !important
    #id {property:value !important;} This rule is quite strong!
    #id {property:value !important;} However, this rule is even stronger, as it overrides its twin above. CSS is read from top to bottom and in conflicts, last declared rule wins.

    PROPERTY:VALUE; EFFECT
    position: static; This is the default property of elements. Top, bottom, left, and right properties will not work in this case.
    position: fixed; Fixed elements render relatively to the browser window, and will not move even if the window is scrolled. Top, bottom, left, and right properties can be used in this case. These elements can overlap other elements, as they are ignored from the normal flow.
    position: relative; A relative positioned element is positioned relative to its normal position.
    position: absolute; Absolute elements can overlap other elements.
    float: left; An element gets pushed to the left (or right), allowing other elements to wrap around it.
    float: none; This ensures elements will not float.
    visibility: hidden; This propery hides an element but it's position remains, affecting the layout as if it was there.
    display: none; The element becomes hidden, as if it is not there.
    display: inline; The element takes up as much width as necessary and doesn't force line breaks.
    display: block; The element behaves as a rectangular box with the fullest width available.
    display: flex; Use to position elements in a flexible way.
    flex-direction: row; Use to position elements in a row (or column).
    flex-direction: row-reverse; Use to position elements in a reversed row (or reversed column).
    align-items: flex-start | flex-end | center | baseline | stretch | space-between | space-around; Controls alignment of all items on the cross axis.
    align-content: flex-start | flex-end | center | stretch | space-between | space-around | space-evenly; Used for controlling the space between flex lines on the cross axis.
    align-self Controls alignment of an individual flex item on the cross axis.
    justify-content: start | center | end | left | right | normal | space-between | space-around | space-evenly | strech | flex-start | flex-evenly; Controls alignment of all items on the main axis.
    flex-wrap: nowrap | wrap | wrap-reverse; Wrap a row or column.
    flex-shrink: 2; Use to shrink items. The value is a number.
    flex-grow: 3; Use to enlarge items. The value is a number.
    flex: basis: 10em; Specifies the initial size of the item before CSS makes adjustments with flex-shrink or flex-grow. The units used by the flex-basis property are the same as other size properties (px, em, %, etc.). The value auto sizes items based on the content.
    order: 1; Use to order items. The value is a number.
    display: grid; Grid, the parent element is referred to as the container and its children are called items.
    grid-template-columns: 50px 50px; This will give your grid two columns that are 50px wide each.
    grid-template-rows: 100px 100px 100px; This will give your grid three rows that are 100px tall each.
    grid-template-columns: auto 50px 10% 2fr 1fr; This snippet creates five columns. The first column is as wide as its content, the second column is 50px, the third column is 10% of its container, and for the last two columns; the remaining space is divided into three sections, two are allocated for the fourth column, and one for the fifth.
    grid-column-gap: 10px; This creates 10px of empty space between all columns.
    grid-row-gap: 10px; This creates 10px of empty space between all rows.
    grid-gap: 10px 20px; Introduce a 10px gap between the rows and 20px gap between the columns.
    grid-column: 1/3; This will make the item start at the first vertical line of the grid on the left and span to the 3rd line of the grid, consuming two columns.
    grid-row: 2/4; This will make the item consume the last two rows.
    justify-self Align the content's position within its cell horizontally.
    justify-items: stretch | center | start | end; Align all grid items's position horizontally.
    grid-template-areas: 
                            "a a ."
                        "a a ."
                        ". b c";
    		 After creating an areas template for your grid container you can place an item in your custom area by referencing the name you gave it.
    grid-area: 3/1/4/4; Place the element with between the third and fourth horizontal lines and between the first and fourth vertical lines.
    grid-template-rows: repeat(100, 50px); Create a 100 row grid, each row at 50px tall.
    grid-template-columns: 100px minmax(50px, 200px); Create two columns; the first is 100px wide, and the second has the minimum width of 50px and the maximum width of 200px.
    repeat(auto-fill, minmax(60px, 1fr)); When the container changes size, this setup keeps inserting 60px columns and stretching them until it can insert another one.

    DOM (Document Object Model)

    FUNCTION EFFECT
    document.body.innerHTML = 'Hello, there!'; A body is an element of the DOM, we can access it using the document object and change the content of the innerHTML property.
    document.getElementById('idName') Returns just one element (every element should have a unique ID), so there should only ever be one element to return.
    document.querySelector('#square') It takes any CSS selector and queries the contents of the DOM. It returns one result.
    document.querySelectorAll('.red') Returns all elements with the class red as a NodeList. This looks very similar to an array, but you won't be able to use array methods on it such as map and reduce. You can access each item using an index, e.g. myNodeList[0], and you can iterate over it using a for loop or using forEach().
    document.getElementsByClassName('className') Returns an array of all elements with a certain class name.
    document.getElementsByTagName('tagName') Returns an array of all elements with a certain tag name.
    document.querySelector('#square').classList.remove('blue'); Finds an element and removes a specific class if it exists, otherwise it returns undefined.
    document.querySelector('#square').classList.add('red'); Finds an element and adds a specific class if it exists, otherwise it returns undefined.
    document.createTextNode('Some new text'); Creates new text node.
    element.appendChild(newNode); Adds a new child node to an element as the last child node.
    element.insertBefore(node1, node2); Inserts node1 as a child before node2.
    element.removeChild(childNode); Removes a child node from a parent element.
    element.replaceChild(newNode, oldNode); Replaces a node with a new one.
    document.createElement(element); Creates new element node.
    element.cloneNode(); Clones an element and returns the resulting node.
    document.getElementById('myImg').src = 'newImg.png'; Finds an element by id and changes the src attribute.
    document.getElementById('myLink').href = 'https://www.github.com'; Finds an element by id and changes the href attribute.
    document.getElementById('myParagraph').style.color = 'red'; Finds an element by id and changes the color attribute.
